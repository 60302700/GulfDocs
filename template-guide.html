Build Your Own Document Template

Custom Template lets you drop in any HTML file and GulfDocs will fill it with your business, client and line-item data. This guide explains exactly how the templating engine works and why placeholders are the piece that makes it all possible.

? Why placeholders matter

A placeholder is a small marker — written as {{something}} — that you leave inside your raw HTML wherever a real value should eventually appear. You never type "Al Noor Trading LLC" or today's invoice total directly into the template; instead you write {{company_name}} and {{grand_total}}, and the engine swaps them out at render time.

What this buys you

One template, unlimited documents. The HTML file you upload is a reusable mold — fill the sidebar form once and every placeholder updates live in the preview, without touching the template again.

Design stays separate from data. You control fonts, colors, spacing and layout freely in plain HTML/CSS. The engine only ever touches the exact substrings that match a known placeholder — everything else is left completely alone.

Nothing leaves your browser. Because rendering happens by doing plain text substitution on the template you supplied, no data is uploaded to any server — the whole process runs client-side.

If you skip placeholders and hard-code values instead, your template will just render the same static text every time — the form fields on the left will have nothing to control. Placeholders are what turn a static HTML page into a data-driven document.

1 Getting started

A custom template is just a normal, self-contained HTML file. Follow these steps to build one:

1. Start from a full HTML document

Include <!DOCTYPE html>, a <head> with your <style> block, and a <body> with your layout. Everything is inline CSS or a <style> tag — the template is rendered inside an isolated iframe/print window, so no external stylesheet other than what you embed will load.

2. Drop placeholders where data should go

Anywhere inside your markup — inside text, an attribute like src="{{company_logo}}", even inside a table cell — type the placeholder tag from the reference table below.

3. Upload & preview

Go to the Template Editor, open the Template & Syntax tab, and drag your .html file onto the drop zone (or click it to browse). Switch back to Document Data and start typing — the live preview on the right updates on every keystroke.

4. Export as PDF or save as a profile

Use Export PDF to print the finished document. Use Export JSON to save your business details, client info and the custom template itself together in one file — Import Profile restores all three in one click next time.

2 Placeholder reference

These are every placeholder the engine understands. Anything typed as {{anything_else}} that isn't on this list is simply removed from the final output, so double-check spelling against this table.

Business / Sender

Tag Description
{{company_logo}} Base64 image data — use inside an <img src="{{company_logo}}"> tag.
{{company_name}} Your business name.
{{company_trn}} Tax Registration / TRN / VAT number.
{{company_phone}} Business phone number.
{{company_email}} Business email address.

Document meta

Tag Description
{{document_title}} e.g. "TAX INVOICE" or "QUOTATION".
{{document_number}} Reference / document number.
{{document_date}} Issue date, formatted (e.g. "14 July 2026").
{{document_due_date}} Due date, formatted the same way.

Client / bill to

Tag Description
{{client_name}} Client / customer name.
{{client_trn}} Client TRN / VAT registration.
{{client_phone}} Client contact phone.
{{client_email}} Client email address.
{{client_address}} Client billing / shipping address.

Financials & totals

Tag Description
{{currency}} Selected currency code (QAR, AED, SAR, OMR, BHD, KWD, USD).
{{subtotal}} Sum of all line items before VAT/discount.
{{vat_rate}} The tax rate, as a plain number (e.g. 5).
{{vat_amount}} Calculated VAT value.
{{discount}} Discount amount (blank/empty when zero — pair with {{#if discount}}).
{{grand_total}} Subtotal + VAT − discount.

Footer

Tag Description
{{notes}} Terms, conditions or footnote text.

3 Conditional blocks

Wrap a section of HTML in {{#if field}} ... {{/if}} to only render it when that field has a value, or {{#unless field}} ... {{/unless}} to render it only when the field is empty. This is how the default template hides an entire "VAT/TRN" line when no TRN was entered, instead of printing "VAT/TRN: " with nothing after it.

{{#if company_trn}}
<div class="trn-box">VAT/TRN: {{company_trn}}</div>
{{/if}}
{{#if}} and {{#unless}} work with any top-level field name from the reference table (not just company_trn) — e.g. document_due_date, client_address, discount. Inside an {{#items}} loop, only {{#if item_sku}} / {{#unless item_sku}} are supported.

4 Repeating line items

Line items are a list, so they need a loop instead of a single placeholder. Wrap your row markup in {{#items}} ... {{/items}} — the engine repeats everything inside it once per line item you've added in the sidebar.

<table>
  <tbody>
    {{#items}}
    <tr>
      <td>{{item_index}}</td>
      <td>{{item_description}}</td>
      <td>{{item_sku}}</td>
      <td>{{item_qty}}</td>
      <td>{{item_price}}</td>
      <td>{{item_total}}</td>
    </tr>
    {{/items}}
  </tbody>
</table>
Tag Description
{{item_index}} 1-based row number.
{{item_description}} Product / service description.
{{item_sku}} SKU or item code.
{{item_qty}} Quantity.
{{item_price}} Unit price, fixed to 2 decimals.
{{item_total}} Qty × price, fixed to 2 decimals.

In a hurry? Skip building your own table and just drop in {{items_table}} — it's replaced with a complete, pre-styled items table (with a "no items added" empty state) so you don't need to write the loop yourself.

5 Theming

GulfDocs's theme manager (the color palette in the sidebar) injects five CSS custom properties into your template automatically, right before </head>. Reference them in your own <style> block and your template will repaint live as the user changes colors — you don't need to hard-code hex values.

.doc-meta h1 { color: var(--doc-primary); }
body { color: var(--doc-text); background: var(--doc-surface); }
.company-info p { color: var(--doc-text-secondary); }
table { border-color: var(--doc-border); }

If a variable isn't set (template opened outside GulfDocs, or ThemeManager unavailable) it's safe to give it a plain CSS fallback, e.g. color: var(--doc-primary, #2563eb);

! Common pitfalls

Unmatched placeholders disappear silently

Any leftover {{...}} that isn't a recognized tag — a typo like {{client_naem}} — is stripped out of the final HTML rather than shown as an error. If a value seems to be missing, re-check the spelling against the reference table above first.

Placeholders only substitute exact tag names

There's no support for nested object paths (e.g. {{client.name}}) or custom/arbitrary field names — only the flat list in the reference table above and the item fields inside an {{#items}} block are understood.

Always close what you open

Every {{#if}}, {{#unless}} and {{#items}} block needs its matching {{/if}}, {{/unless}} or {{/items}} — an unclosed block will swallow the rest of your document.

Your template never leaves the browser

Substitution happens entirely on the client — nothing is uploaded. That also means the template only exists for your current session unless you save it: use Export JSON in the editor to keep a permanent copy alongside your business profile.

6 Minimal starter template

Copy this into a .html file and upload it to try the full placeholder system end to end.

<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>{{document_title}}</title>
  <style>
    body { font-family: sans-serif; padding: 32px; color: var(--doc-text); }
    h1 { color: var(--doc-primary); }
    table { width: 100%; border-collapse: collapse; margin-top: 20px; }
    td, th { border-bottom: 1px solid var(--doc-border); padding: 8px; text-align: left; }
  </style>
</head>
<body>
  <h1>{{company_name}}</h1>
  {{#if company_trn}}
  <p>VAT/TRN: {{company_trn}}</p>
  {{/if}}

  <h2>{{document_title}} — {{document_number}}</h2>
  <p>Bill to: <strong>{{client_name}}</strong></p>

  <table>
    <thead><tr><th>#</th><th>Description</th><th>Qty</th><th>Total</th></tr></thead>
    <tbody>
      {{#items}}
      <tr>
        <td>{{item_index}}</td>
        <td>{{item_description}}</td>
        <td>{{item_qty}}</td>
        <td>{{item_total}}</td>
      </tr>
      {{/items}}
    </tbody>
  </table>

  <p><strong>Grand Total: {{currency}} {{grand_total}}</strong></p>
  <p>{{notes}}</p>
</body>
</html>
